Detalle consulta tarjeta débito

Mediante esta opción que se invoca mediante el hipervínculo Número de tarjeta contenido en la consulta de clientes, la entidad tienen acceso a toda la información de cada una de las tarjetas débito de sus clientes como lo son cuentas asociadas, histórico de movimientos, estados de la misma entre otros datos.

Al ingresar a la opción se despliega un formulario estructurado en cuatro tabs en los cuales se agrupa toda la información de la tarjeta: Información general, Cuentas, Movimiento y Novedades. Por defecto el formulario siempre ingresa en el tab de información general.

 

Información general: si el usuario invoca el tab Cuentas, se despliega un nuevo formulario que contiene la información principal de la tarjeta como son la identificación del producto, bloqueos actuales y anteriores, fechas importantes. El formulario se encuentra estrucuturado en tres bloques de datos: Información principal, Generalidades y Bloqueos.

 

 

Descripción de campos

Nombre

Campo que despliega el nombre y apellidos del cliente titular de la tarjeta.

Empresa

En este campo se muestra el nombre de la empresa o entidad emisora de la tarjeta.

Sucursal

Campo que contiene la sucursal de radicación de la tarjeta consultada.

Tipo tarjeta

Este campo indica si el tipo de tarjeta consultada corresponde a débito.

Agente

Campo que muestra el nombre del agente responsable de la colocación de la tarjeta.

Producto

En este campo se despliega el nombre del producto débito al que pertenece la tarjeta.

Ciclo

Campo en el que se indica el ciclo de facturación al que pertenece la tarjeta y que determina la fecha de corte para el cobro de la cuota de manejo.

Codigo producto (segmento)

Este campo contiene el código del segmento al que pertenece la tarjeta de acuerdo con el producto débito.

Grupo de afinidad

Campo que muestra el grupo de afinidad correspondiente al producto débito al que pertenece la tarjeta.

Tarjeta anterior

En este campo se muestra si es el caso el anterior número que tenía la tarjeta cuando por ejemplo a causa de las reexpediciones, reposiciones o renovaciones de la misma, ésta haya tenido un cambio de número.

Fecha última novedad

Campo que muestra en formato YYYY-MM-DD la fecha en la cual se le aplicó a la tarjeta la novedad más reciente, en caso de existir.

Fecha vencimiento

Este campo despliega en formato YYYY-MM-DD la fecha hasta la cual tiene vigencia la tarjeta consultada.

Fecha activación

Campo que muestra en formato YYYY-MM-DD la fecha en la cual se activó la tarjeta actual.

Fecha cuota

Este campo señala en formato YYYY-MM-DD la próxima fecha en la que se le debe generar el cobro de cuota de manejo a la tarjeta.

Fecha manejo anterior

Campo en formato YYYY-MM-DD que indica la fecha en la cual se le hizo el último cobro de cuota de manejo a la tarjeta.

Valor manejo

En este campo se muestra el monto a cobrar a la tarjeta por concepto de cuota de manejo de acuerdo con lo parametrizado para el producto.

Bloqueo persona / Fecha bloqueo persona

Campos que muestran respectivamente en bloqueo actual que tenga el cliente titular de la tarjeta y la fecha en la cual este fué aplicado. Si el campo Bloqueo está en blanco y la Fecha si tiene dato se debe interpretar como la fecha en que se levantó el bloqueo al cliente.

Bloqueo anterior persona / Fecha bloqueo anterior persona

Si en alguna ocasión anterior el cliente tuvo algún bloqueo, en estos campos se muestra el bloqueo y la fecha de aplicación / retiro del mismo.

Bloqueo plástico / Fecha bloqueo plástico

Campos que muestran respectivamente en bloqueo actual que tenga la tarjeta consultada y la fecha en la cual este fué aplicado. Si el campo Bloqueo está en blanco y la Fecha si tiene dato se debe interpretar como la fecha en que se levantó el bloqueo a la tarjeta.

Bloqueo anterior plástico / Fecha bloqueo anterior plástico

Si en alguna ocasión anterior la tarjeta tuvo algún bloqueo, en estos campos se muestra el bloqueo y la fecha de aplicación / retiro del mismo.

Motivo cancelación

En caso que la tarjeta se encuentre con bloqueo de cancelación voluntaria, en este campo se muestra el código y descripción del motivo presentado por el cliente al momento de solicitar la cancelación de la tarjeta.

 

Cuentas: si el usuario invoca el tab Cuentas, se despliega un nuevo formulario que contiene la información relacionada con todas las cuentas asociadas a la tarjeta consultada.

Este formulario contiene los hipervínculos Adicionar y eliminar cuenta, Cambio categoría cuenta, Cambio de cuenta, que le permiten invocar las novedades correspondientes para modificar la información de las cuentas.

 

Descripción de campos

Número de cuenta

Campo que muestra el número de cada una de las cuentas asociadas a la tarjeta

Tipo de cuenta

En este campo se indica si la cuenta es de Ahorros, Corriente o Rotativo

Moneda

Campo que despliega la moneda en la cual se encuentra denominada cada una de las cuentas asociadas a la tarjeta.

Secuencia de cuenta

En este campo se muetra la secuencia interna que el sistema le asigna a cada una de las cuentas adicionadas a la tarjeta.

Código estado de cuenta

Campo que indica si la cuenta se encuentra Activa o Inactiva

Fecca estado cuenta

En este campo se muestra en formato YYYY-MM-DD la fecha a la que corresponde el estado actual de la cuenta.

Categoría de cuenta

Campo que indica si la cuenta el Primaria (principal) o Secundaria.

 

Movimiento: si el usuario invoca el tab Movimiento, se despliega un nuevo formulario que contiene la información relacionada con todas las transacciones efectuadas con la tarjeta consultada, bien sea que las haya efectuado el cliente o que generadas de manera automática por el sistema.

 

 

Descripción de campos

Secuencia movimiento

Número utilizado por las entidades para identificar el orden de ingreso de movimiento de varios orígenes dentro del sistema

Fecha Comprobante

En este campo se indica la fecha que se realizó la transacción.

Numero Comprobante

En este campo muestra el numero asignado para el comprobante de la transacción.

Transacción interna

En este campo se indica la transacción interna generada por el sistema al momento de aplicar la transacción al cliente.
Establecimiento Campo que contiene el codigo y el nombre del comercio donde se realizó la transacción.

Valor total

Campo que contiene el monto por el cual fue registrada y aplicada la transacción en el sistema.

 

El formulario contiene la opción Detalle.

Detalle: si el usuario selecciona alguna transacción e invoca la opción detalle, se despliega un formulario que le muestra todos los datos de la transacción.

 

Descripción de campos

Cliente

Campo que despliega el nombre y apellidos del cliente titular de la tarjeta

Número de tarjeta

Muestra la tarjeta que realizó la transacción.

Secuencia movimiento

Número utilizado por las entidades para identificar el orden de ingreso de movimiento de varios orígenes dentro del sistema

País

En este campo indica en que país realizó la transacción

Ciudad

En este campo indica en que país realizó la transacción

Origen

Despliega el nombre de la fuente y origen del movimiento a través del que ingresa la transacción

Numero Comprobante

En este campo muestra el numero asignado para el comprobante de la transacción.

Fecha Comprobante

En este campo se indica la fecha que se realizó la transacción.

Código autorización

Campo que indica el número de autorización otorgado poir el sistema en el momento de aprobar la transacción.

Valor moneda local

Este campo despliega el valor de la transaccion en el país de origen.

Valor compra

Este campo despliega el valor de la transaccion sin tener en cuenta propina e iva.

Valor propina

Este campo despliega el valor de la propina dado por el tarjehabiente al establecimiento.

Valor iva

Este campo despliega el valor del impuesto del valor agregado a las ventas

Valor total

Este campo despliega el valor total de la transacción

Valor base reintegro iva

Este campo despliega el valor que sirve como base para el calculo del reintegro de los 2 puntos de IVA sobre compras y que corresponde al valor de los bienes y/o servicios gravados con este impuesto.

Establecimiento

En este campo se muestra el código del establecimiento en el que se realizó la transacción.

Transacción interna

En este campo se indica la transacción interna generada por el sistema al momento de aplicar la transacción al cliente.

Moneda original

Campo que indica la moneda en la cual fue efectuada la transacción.

Moneda local

Campo que indica la moneda en la cual fue efectuada la transacción y que corresponde a la misma en la cual están determinados los saldos de la cuenta a afectar como resultado de las utilizaciones de la tarjeta.

Fecha proceso

En este campo se muetra en formato YYYY-MM-DD la fecha en que se procesó el movimiento dentro del cual llegó la transacción

Bin fuente

Campo que contiene el bin que identifica a la entidad que emite la tarjeta.

Fecha contabilización

Este campon en formato YYYY-MM-DD contiene la fecha en la que el movimiento fue contabilizado.

Número paquete

Señala el número utilizado por las entidades para identificar el orden de ingreso o captura de varios paquetes de movimiento dentro del sistema.

Lote referencia

Este campo muestra el número del lote dentro del cual se encuentra la transacción.

Secuencia referencia

Este campo corresponde a un número secuencial que hace parte de la referencia universal que identifica cada transacción.

Dígito de chequeo

Este campo contiene el último número que compone la referencia universal, obtenido mediante algoritmos y que permite verificar la validez de dicha referencia

Referencia internacional

Campo que contiene el número de referencia universal asignado a la transacción y que la identifica dentro de las demás transacciones.

Tipo captura

En este campo se indica la manera como la transacción ingresó al sistema.

 

Novedades: si el usuario invoca el tab Novedades, se despliega un nuevo formulario que contiene la información relacionada con todas las novedades que le han sido aplicadas a la tarjeta. Contiene la opción Detalle que permite invocar el formulario con la información completa de cada una de las novedades aplicadas a la tarjeta.

 

Descripción de campos

Secuencia

Campo que muestra el número asignado internamente por el sistema a cada una de las novedades generadas.
Fecha Novedad En este campo se muetra en formato YYYY-MM-DD la fecha en que se procesó la novedad.

Novedad adminsitrativa

En este campo se despliega el código y nombre de la novedades que fue aplicada a la tarjeta.

Clase novedad

Campo en el que se indica si la acción de la novedad fue Adición, Modificación o Eliminación de datos.
