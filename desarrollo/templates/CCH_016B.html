Actualización / Manejo de actividades

Una vez se ha ejecutado el proceso de Actividades por agente de cobro se alimenta la mayor parte de los campos de la forma o pantalla, con la información básica y necesaria para que cuando cada agente ingrese al sistema le sean reportadas las obligaciones que se le han asignado. A partir de este punto, la forma permite la interacción con algunos de sus campos para registrar la evolución de cada una de las actividades.

La opción consta de un wizard de dos formularios:

Actividades:

 

Resumen: Si el usuario invoca la opción Adicionar se despliega un formulario con los siguientes campos:

Actividad

Dupla de campos que despliegan el código y nombre de la actividad asociada al cobro, de acuerdo con lo parametrizado en la opción Actividades de cobro de este mismo módulo.

Número de obligación

Campos de salida en los que son desplegados el código que identifica el tipo de obligación y número de la misma.

Tipo de cobro

En estos campos de salida se presentan el código del tipo de cobro.

Número de cobro

Muestra el número de la cuota sobre la que se adelantan las gestiones de cobro, donde por defecto aparecerá el número 1; este valor podrá variar señalando el número de cada una de las cuotas de la misma obligación, a las que se les esté adelantando proceso de cobro.

Prioridad

Campo que sirve como referente para determinar la jerarquía en que deben ser atendidas las actividades.

Puntaje

Indica el valor porcentual que junto con el campo Prioridad brindan elementos para la jerarquización de la atención de las actividades asignadas a cada agente de cobro. Dicho valor porcentual corresponde a la parametrización efectuada en la opción  Parámetros de calificación de este mismo menú.
