Plan de pago preferencial

Mediante esta funci�n se habilita un centro de autorizaciones manual, con el fin de atender los establecimientos a los cuales les es permitido efectuar transacciones de pago preferencial.

Al ingresar al formulario el usuario debe registrar el número de tarjeta, el código alfa, el valor de la transacción y el código del establecimiento que solicita la autorización.

 

 

Número tarjeta

En este campo se captura el número de tarjeta del cliente a quien se le otorgará la autorización.

Código alpha

Campo obligatorio en el que se registra el número del código alpha de la tarjeta a autorizar.

Valor

En este campo obligatorio se ingresa el monto de la transacción que se está solicitando autorizar.

Transacción

Campo de salida no modificable que muestra la transacción original con la cual el autorizador reconoce que es una transacción Plan de Pago Preferencial para que pueda afectar directamente la línea PPP y no la de consumos de la tarjeta.

Establecimiento

En este campo obligatorio se debe ingresar el código del establecimiento que solicita la autorización. Al momente de cargar la información el sistema debe validar que las posiciones 5 y 6 del código del establecimiento sean 28 o 48 para identificar si el establecimiento está o no autorizado para efectuar transacciones PPP.

Moneda

Campo de salida que muestra la moneda en la cual se denomina la transacción y que por defecto es la moneda patron definida para la entidad.

 

Después de haber diligenciado los datos requeridos en el formulario inicial al utilizar el botón Cargar información una vez que el sistema valide la tarjeta y el establecimiento se despliega un nuevo formulario que muestra toda la información básica de la transacción a autorizar y donde todos los campos son de salida no modificables por el usuario,

 

Número tarjeta

Muestra el número de tarjeta ingresado de la autorización que se dese crear.

Nombre plástico

Despliega el nombre del cliente a quien pertenece la tarjeta.

Tipo identificación

Contiene el tipo documento de identificación del tarjetahabiente.

Número identificación

Indica el número de documento de identificación del tarjetahabiente.

Fecha vencimiento

Muestra la fecha de vencimiento de la tarjeta capturada previamente.

Transacción

Despliega la transacción para la cual se está solicitando la autorización.

Moneda

Contiene la moneda en la cual se denomina la transacción que se está solicitando autorizar.

Valor transacción

Indica el monto por el cual se solicita la autorización

Establecimiento

Despliega el código y nombre el establecimiento que solicita la autorización PPP

Horario

Campo que contiene información del horario en que el establecimiento está autorizado para operar normalmente; este dato ayuda a la entidad a detectar comportamientos fraudulentos con el uso de la tarjeta.

Número identificación

En este campo se muestra el código de identificación del establecimiento en dónde se está realizando la transacción.

Razón social

Muestra el nombre con el cual se encuentra registrado el establecimiento ante la Cámara de Comercio o la entidad que cumpla esa función.

Servicio

Despliega el código y nombre del servicio asociado al comercio a través del cual se realiza la utilización para la que se registra la autorización.

Convenio

Campo de salida que muestra el código y el nombre del convenio al que pertenece la tarjeta cuando es de marca privada

 

Proceso autorización: A través de este botón el sistema inicia el proceso de generar la autorización, y muestra el resultado en el bloque Información de la autorización.

En este bloque, una vez capturada la información requerida para la autorización, se muestra la respuesta arrojada por la solicitud de autorización una vez se apliquen las correspondientes validaciones.

 

Código respuesta

Corresponde al código y descripción de la respuesta a la solicitud de autorización.

Número autorización

En caso de que la solicitud de autorización haya sido exitosa, en este campo aparece el número que fue asignado como aprobación de la misma.

 

Plan de Pago Preferencial. El formulario contiene un hipervínculo denominado Plan de Pago Preferencial que al ser invocado despliega un nuevo formulario que contiene la información requerida por el usuario que está capturando la solicitud de autorización con el fin de permitirle decidir si la niega de manera autónoma o si por el contrario invoca el autorizador para que sea éste quien decida si niega o aprueba la solicitud.

La información desplegada es únicamente de salida y se alimenta mediante el cargue de un archivo. Si para la tarjeta que solicita la autorización no existe información al invocar el hipevínculo se desplegará un mesnaje de error indicando al actor este evento.

Este formulario solo contiene el botón Salir que le permite al actor regresar al formulario inicial desde donde podrá Invocar el autorizador, negar o cancelar la autorización.

OPCIONES ESPECIALES

Negar: mediante este botón el usuario puede negar de manera inmediata la autorización, sin necesidad de invocar el autorizador, cuando después del análisis hecho en el formulario Plan de Pago Preferencial considera que no se debe aprobar la autorización

Proceso autorización: mediante este botón el usuario invoca el autorizador permitiendo que sea éste quien decida con base en las validaciones propias si aprueba o niega la autorización solicitada.

Cancelar: mediante este botón el usuario regresa al formulario inicial de captura y no efectúa ningún tipo de proceso con la autorización.
