Movimiento aplicado a convenios

Mediante esta opción se facilita la consulta de los diferentes movimientos que afectan las cuentas de los convenios, grupos de afinidad o a la de las empresas con las que se han celebrado acuerdos comerciales.

El formulario muestra los campos por los cual se podrá fitrar la información de convenios a consultar. Esta información se puede consultar a través de las opciones Detalle movimiento convenio, Detalle movimiento grupos de afinidad, y Detalle movimiento clientes.

 

Fecha de proceso desde

En este campo en formato YYYY-MM-DD se ingresa la fecha inicial que delimita el periodo para el cual se desea consultar el movimiento aplicado.

Fecha de proceso hasta

En este campo en formato YYYY-MM-DD se ingresa la fecha final que delimita el periodo para el cual se desea consultar el movimiento aplicado.

Esquema de puntos

Campo con lista de valores de la cual se selecciona el Esquema de puntajes, para el cual se requiere consultar el movimiento aplicado, teniendo en cuenta que los programas de puntos han sido asociados a los convenios, en la opción Definición convenios.

 

Adicionar: Si el usuario invoca la opción Adicionar se despliega un formulario con los siguientes campos:

 

Código

Campo alfanumérico de tres dígitos, obligatorio, que contiene el código asignado a cada país y el cual lo identifica a nivel internacional dentro del sistema de tarjeta de crédito.

Nombre

Campo alfanumérico de 35 posiciones, obligatorio, en el cual se registra el nombre oficial de cada país asociado al respectivo código.

Código telefónico

Campo alfanumérico de cuatro posiciones, no obligatorio, en el cual se registra el código telefónico asignado a cada país.

Moneda

Campo no obligatorio, con lista de valores de la cual se selecciona la Moneda emitida por el Banco Central del país en que resida la aplicación.

Reguladora

En este campo no obligatorio, se selecciona de la lista de valores la Moneda patrón, es decir, aquella en la cual se realizan las transacciones económicas, permitiendo además manejar por defecto, todas las cifras existentes dentro del sistema.

Indizadora

Este campo diseñado para un uso futuro, contiene lista de valores de la que se puede seleccionar la moneda puente, o que se va a utilizar para realizar las conversiones requeridas.

Nivel

Campo con lista de valores de la cual se selecciona entre País, Departamento, Municipio y Otro el nivel en el cual se realizará la validación.

Zona postal

En este campo no obligatorio, se señala si la zona postal se valida o no para el país sobre el que se ubica el cursor.

Posición inicial

Campo numérico de 2 dígitos, obligatorio, que indica la posición a partir de la que debe aparecer el dato de la zona postal dentro del registro o cadena de caracteres, cuando se deba validar.

Longitud

En este campo numérico de 2 dígitos, obligatorio, se registra la cantidad o número de posiciones que ocupará el dato de la zona postal dentro del registro o cadena de caracteres.

Región internacional

Campo numérico de 3 dígitos, no obligatorio, permite asociar un código para agrupar varios países acorde con las necesidades de determinadas franquicias.

País internacional

Este campo numérico de 3 dígitos, no obligatorio, permite asociar un código de país diferente al utilizado tanto por Visa como por Master Card, acorde con las necesidades de determinadas franquicias.

 

Actualizar: Si el usuario invoca la opción Actualizar se despliega un nuevo formulario en el cual los únicos campos modificables son: Nombre, Código teléfono, Moneda, Reguladora, Indizadora, Nivel, Zona postal, Posición inicial, Longitud, Región internacional, y País internacional.

 

Detalle: Si el usuario invoca la opción Detalle se despliega el siguiente formulario:

 

Departamentos | Municipios | Otros | Zona postal
